# Test Notes

## Test files created

- `src/main/soniox.integration.test.ts`

## What each test covers

- **"connects, sends audio, and completes the transcription round-trip"**: Verifies the full Soniox pipeline — WebSocket connection, config handshake, audio upload, finalization, and finished signal. If any final tokens are returned, asserts they have non-empty text.

## Acceptance criteria coverage

| Criterion | Covered | Notes |
|-----------|---------|-------|
| Connects to real Soniox WebSocket | Yes | Uses `SonioxClient.connect()` with real API key |
| Sends audio clip | Yes | 1-second 440Hz sine wave PCM |
| Verifies final tokens with non-empty text | Partial | Asserts non-empty text IF tokens are returned; a pure tone may not produce speech tokens |
| Disconnects cleanly | Yes | `afterEach` calls `client.disconnect()` |
| Excluded from `npm test` | Yes (pending config) | Requires vitest config changes |
| Reads API key from env var | Yes | `process.env.SONIOX_API_KEY` |
| Skips gracefully without API key | Yes | `const suite = apiKey ? describe : describe.skip` |
| `test:integration` script | Pending | Requires `package.json` change |
| Documentation | Yes | File header comment |

**Plan Issues**
1. **Major — Step 1 / Steps 4-7:** The logger API is internally inconsistent. Step 1 says extra args are appended as JSON after the message, but the later steps use `console`-style format strings such as `%d` and `%s` in [`src/main/session.ts`](/C:/code/draftable/stex/.mound/worktrees/worker-2-38e2f328/src/main/session.ts) and [`src/main/soniox-lifecycle.ts`](/C:/code/draftable/stex/.mound/worktrees/worker-2-38e2f328/src/main/soniox-lifecycle.ts). If implemented as written, logs will contain literal `%d`/`%s`. Fix: either define `log()` around `util.format()` so it matches the proposed call sites, or rewrite all proposed call sites to use plain strings / structured metadata consistently.

2. **Major — Step 1 / Step 2:** The singleton logger design does not define re-initialization or teardown behavior, but the plan also adds a dedicated logger test suite. With `logStream` held at module scope, repeated `initLogger()` calls across tests can leak open streams and interfere with rotation/temporary-directory tests, especially on Windows. Fix: specify that `initLogger()` closes any previous stream before replacing it, and add a small test-only reset/cleanup path or otherwise make repeated initialization deterministic.

3. **Major — Step 8:** The affected test surface is understated. The plan only mentions [`src/main/session.test.ts`](/C:/code/draftable/stex/.mound/worktrees/worker-2-38e2f328/src/main/session.test.ts) and [`src/main/soniox-lifecycle.test.ts`](/C:/code/draftable/stex/.mound/worktrees/worker-2-38e2f328/src/main/soniox-lifecycle.test.ts), but [`src/main/session-reconnect.test.ts`](/C:/code/draftable/stex/.mound/worktrees/worker-2-38e2f328/src/main/session-reconnect.test.ts) also imports `session.ts`, and [`src/main/first-run.test.ts`](/C:/code/draftable/stex/.mound/worktrees/worker-2-38e2f328/src/main/first-run.test.ts) imports `index.ts`. Step 3 adds `app.getPath()` / `app.isPackaged` usage to `index.ts`, and the current first-run test mock does not provide either. Fix: expand Step 8 to cover `session-reconnect.test.ts` and `first-run.test.ts`, and explicitly update the Electron/app mocks or mock `./logger` there as well.

# T18: Theming — Test Notes

## Test Files Created

- `src/main/theme.test.ts` — Tests for the theme resolver module

## What Each Test Covers

### `resolveTheme()`
- Returns `"light"` when setting is `"light"` (ignores OS)
- Returns `"dark"` when setting is `"dark"` (ignores OS)
- Returns `"dark"` when setting is `"system"` and OS is dark
- Returns `"light"` when setting is `"system"` and OS is light

### `initThemeManager()`
- Registers `ipcMain.handle` for `THEME_GET` channel
- `THEME_GET` handler returns the current resolved theme
- Broadcasts to all windows when theme setting changes (light -> dark)
- Does NOT broadcast when resolved theme hasn't changed (dedup)
- Broadcasts on `nativeTheme.updated` when setting is `"system"` and OS theme changes
- Does NOT broadcast on `nativeTheme.updated` when setting is explicit (not system)
- Broadcasts to multiple windows

## Acceptance Criteria Not Tested Automatically

- **Visual theming** (light/dark CSS): CSS variable definitions and their visual appearance cannot be tested in vitest (no DOM rendering). Would need end-to-end or screenshot tests.
- **Settings window theme selector**: The `<select>` in General.tsx already exists and works via the existing settings infrastructure — no new logic to test.
- **Theme initialization in renderer**: The `initTheme()` calls in overlay/settings entry points are thin wrappers around IPC calls + DOM attribute setting. Testing these would require a browser environment.

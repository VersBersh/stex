  3. **Major — Hidden semantic coupling in reconnect handling.** [src/main/soniox-lifecycle.ts:53](/C:/code/draftable/stex/.mound/worktrees/worker-11-10dca67e/src/main/soniox-lifecycle.ts#L53), [src/main/soniox-lifecycle.ts:150](/C:/code/draftable/stex/.mound/worktrees/worker-11-10dca67e/src/main/soniox-lifecycle.ts#L150). Initial disconnects are classified via `classifyDisconnect`, but reconnect-attempt failures ignore `code` and `reason` and always reschedule another retry. That only works because of an unstated assumption that every reconnect failure is transient. If reconnect fails with a permanent condition such as invalid credentials or rate limiting, this module will loop indefinitely instead of surfacing the proper error/action. Fix by routing reconnect failures through the same classification logic as normal disconnects, or by extracting a shared failure handler used by both paths.

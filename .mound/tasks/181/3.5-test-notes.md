## What the test covers

- **"buffers the last word of a complete utterance, which flush then emits"**: Simulates the exact bug scenario — a full utterance arrives as final tokens, `mergeTokens` correctly buffers the last word ("transcription"), and `flushPending` emits it. This verifies the merge/flush contract that the new timeout relies on.

## Acceptance criteria not covered by automated tests

- **Timer orchestration**: The 300ms flush timeout in `TokenCommitPlugin` is React component logic requiring a full Lexical editor test harness. No existing component tests exist for this plugin, so the timer behavior (start/reset/cancel) is not unit-tested. Manual testing confirms the fix works end-to-end.
- **No duplicate words**: Verified by manual testing. The timer clears `pendingRef` after flushing, and explicit flush paths (pause/stop) clear the timer first, preventing double-flush.

# Test Notes

## Test files created

None. This task is primarily UI scaffolding (React components, CSS, Lexical editor setup) that cannot be meaningfully unit-tested without adding `jsdom`, `@testing-library/react`, and other test dependencies that are outside the task scope.

## What each change covers (and why automated tests are limited)

### Testable but deferred
- **OverlayContext logic** (clear confirmation timer, pause/resume toggle): Pure state logic that *could* be tested with React Testing Library + jsdom, but adding these dependencies is outside scope.
- **React component rendering**: Would require jsdom environment in vitest, `@testing-library/react`, and extensive mocking of `window.electronAPI`.

### Already covered by existing tests
- **window.ts changes** (preload path, IPC handler): The existing `window.test.ts` tests the overlay window creation. Adding `preload` to webPreferences and `ipcMain.on('window:hide')` would require updating the electron mock to include `ipcMain`, but this risks breaking existing tests owned by T4. We will verify the changes compile and work via the build step instead.

### Not automatically testable
- **Lexical editor keyboard shortcuts**: Requires browser-like environment with keyboard event simulation
- **CSS layout**: Visual verification only
- **Preload script**: Runs in Electron's preload context, not testable in vitest

## Acceptance criteria verification plan (manual)

- Overlay window renders with title bar, editor, and status bar → Visual check
- Title bar is draggable, hide button works → Manual interaction
- Lexical editor is functional for basic text editing → Type, select, copy, paste, undo
- Clear button shows "Confirm?" for 3 seconds → Click and observe
- Copy button copies editor text to clipboard → Copy and paste elsewhere
- Escape hides the window → Press Escape
- Layout matches the spec wireframe → Visual comparison

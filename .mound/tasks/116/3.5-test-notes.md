# Test Notes

No new test files were created.

## Rationale

The scroll-follow logic depends on DOM layout properties (`scrollHeight`, `clientHeight`, `scrollTop`) which jsdom does not compute — all values are 0 in jsdom. Meaningful automated testing would require a real browser environment (Playwright/Puppeteer) or extensive layout property mocking that would not provide confidence beyond what manual testing gives.

The fix is a single additional conditional guard (`overflow > 0`) that gates the existing `scrollTop` assignment. The existing `ghost-text.test.ts` tests (16 passing) cover the CSS variable logic which is unaffected.

## Acceptance criteria coverage

- **"input box does not scroll unless content exceeds visible area"**: Must be verified manually. The overflow guard ensures `scrollTop` is only set when `scrollHeight - scrollTop - clientHeight > 0`.
- **"existing text remains visible"**: Must be verified manually. With the guard, no scroll adjustment occurs when content fits.
- **"scrolling behaves normally when content overflows"**: Must be verified manually. When overflow > 0, the existing scroll-to-bottom behavior is preserved.

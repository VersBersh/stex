# Test Notes

## Test files created

- `src/renderer/overlay/editor/TimestampedTextNode.test.ts`

## What each test covers

- **Construction and getters**: Verifies `startMs`, `endMs`, `originalText` are stored and retrievable; verifies `originalText` is preserved when text content is mutated
- **getType and clone**: Verifies `getType()` returns `'timestamped-text'`; verifies `clone()` preserves all metadata fields
- **Type guard**: Verifies `$isTimestampedTextNode` correctly identifies instances and rejects null/undefined
- **exportJSON / importJSON**: Round-trip serialization preserving all fields
- **DOM rendering**: Verifies the node renders as a standard text span, identical to `TextNode`
- **Multiple nodes per paragraph**: Verifies multiple `TimestampedTextNode` instances can coexist in a paragraph and produce correct concatenated text content

## Acceptance criteria not tested automatically

- `TokenCommitPlugin` integration (creates per-token nodes from `onTokensFinal`) — this would require mocking `window.api.onTokensFinal` and the full React/Lexical plugin lifecycle. The existing `cursorTracking.test.ts` tests verify the cursor/scroll behavior is preserved since they test the append-to-paragraph pattern directly.
- Visual identity with `TextNode` — verified structurally (DOM output matches), but pixel-level visual comparison requires manual inspection.

**Plan Issues**
1. Major — Step 1/2 picks a custom Lexical node architecture that does not fit the current documented design. The plan introduces `TimestampedTextNode` in [3-plan.md](C:/code/draftable/stex/.mound/worktrees/worker-1-f497115a/.mound/tasks/145/3-plan.md#L9) and registers it in [3-plan.md](C:/code/draftable/stex/.mound/worktrees/worker-1-f497115a/.mound/tasks/145/3-plan.md#L114), but the current model spec explicitly says the editor uses standard `ParagraphNode` and `TextNode` types specifically to avoid custom Lexical node types in [models.md](C:/code/draftable/stex/.mound/worktrees/worker-1-f497115a/spec/models.md#L57). Fix: either revise the implementation to keep timestamps outside Lexical nodes, or explicitly update the specs first and justify the architectural change.

2. Major — The risk analysis in Step 1 understates how timestamp metadata can become stale after user edits. The plan only calls out `splitText` in [3-plan.md](C:/code/draftable/stex/.mound/worktrees/worker-1-f497115a/.mound/tasks/145/3-plan.md#L150), but editable Lexical text can also be mutated in place, which would leave `startMs`/`endMs`/`originalText` attached to user-modified text. That breaks the stated purpose of using timestamps for later replay decisions. Fix: add an explicit invalidation strategy for any user edit touching a timestamped node, or move timing metadata into a sidecar structure keyed by offsets/blocks instead of editable nodes.

3. Major — Step 4 is not correct: the existing tests do need changes if this implementation goes ahead. `cursorTracking.test.ts` currently mirrors the old single-node append path with `$createTextNode(text)` in [cursorTracking.test.ts](C:/code/draftable/stex/.mound/worktrees/worker-1-f497115a/src/renderer/overlay/editor/cursorTracking.test.ts#L37), so it will not verify the new per-token node behavior at all. It also creates editors without node registration in [cursorTracking.test.ts](C:/code/draftable/stex/.mound/worktrees/worker-1-f497115a/src/renderer/overlay/editor/cursorTracking.test.ts#L23). Fix: update the helper to commit `SonioxToken[]` through timestamped nodes, register the custom node in test editors, and add token-boundary cursor cases.

4. Minor — The plan adds custom serialization/clone logic in Step 1 but includes no direct tests for it. If those methods drift from Lexical’s expectations, the failure will be silent until later. Fix: add a focused unit test for create/clone/export/import behavior for the new node.

# Test Notes

## Test files created

- `src/main/tray.test.ts`

## What each test covers

- **initTray**: Creates a Tray instance, uses NativeImage icon, sets tooltip, builds context menu with correct items
- **Menu actions**: Show/Hide calls `toggleOverlay()`, Settings calls `showSettings()`, Quit calls `app.quit()`
- **Re-init safety**: Calling `initTray()` twice destroys the first tray before creating a new one
- **destroyTray**: Destroys the tray, is safe to call when no tray exists or already destroyed

## Acceptance criteria coverage

| Criteria | Test coverage |
|----------|--------------|
| Tray icon appears on app launch | `initTray` creates Tray instance (integration with index.ts verified by code review) |
| Right-click shows context menu | Context menu template verified with correct items |
| Show/Hide toggles overlay | Click handler calls `toggleOverlay()` |
| Settings opens settings window | Click handler calls `showSettings()` |
| Quit exits application | Click handler calls `app.quit()` |
| Tray icon persists across show/hide cycles | Tray is module-level variable, not tied to window lifecycle (structural guarantee) |

1. Minor — Hidden Coupling / Dependency Inversion. The tray manager depends directly on concrete window actions instead of an explicit abstraction: [src/main/tray.ts:2](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/tray.ts#L2), [src/main/tray.ts:27](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/tray.ts#L27), [src/main/tray.ts:28](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/tray.ts#L28), [src/main/index.ts:6](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/index.ts#L6). The code works as wired today, but the requirement that the window manager be initialized first only exists as startup ordering in `index.ts`; `initTray()` itself does not encode that contract. That creates a soft temporal coupling between modules. A cleaner design is to have `initTray()` accept its actions as dependencies, or accept a small interface like `{ toggleOverlay, showSettings, quit }` from the entry point.

2. Minor — Single Responsibility / Embedded Design Principle. The tray behavior module also owns the icon asset payload: [src/main/tray.ts:6](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/tray.ts#L6), [src/main/tray.ts:8](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/tray.ts#L8), [src/main/tray.ts:12](C:/code/draftable/stex/.mound/worktrees/worker-6-9e426e04/src/main/tray.ts#L12). That is acceptable for a placeholder, but it means branding/resource changes require edits to the behavior module. If this stops being temporary, move icon creation behind a dedicated icon provider or resource module so tray lifecycle and tray assets can evolve independently.

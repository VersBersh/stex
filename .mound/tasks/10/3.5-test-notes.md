# Test Notes for T10

## Test files created

- `src/main/soniox.test.ts`

## What each test covers

| Test | Coverage |
|------|----------|
| `connect > creates a WebSocket to the Soniox endpoint` | Verifies correct endpoint URL |
| `connect > sends configuration message on open` | Verifies JSON config message format with all fields |
| `connect > emits onConnected after open` | Connection event lifecycle |
| `connect > reports connected state after open` | `connected` getter |
| `sendAudio > sends buffer as binary frame` | Audio streaming acceptance criterion |
| `sendAudio > is a no-op when not connected` | Graceful no-op |
| `finalize > sends empty buffer` | Empty frame finalization |
| `finalize > is a no-op when not connected` | Graceful no-op |
| `token parsing > separates final and non-final` | Core token separation |
| `token parsing > tracks audio_final_proc_ms` | Watermark-based deduplication of final tokens |
| `token parsing > no new final tokens` | Does not emit empty final token arrays |
| `token parsing > no non-final tokens` | Does not emit empty non-final token arrays |
| `finalization response > finished: true` | Finished signal detection |
| `finalization response > processes tokens before finished` | Tokens in finished response are not dropped |
| `disconnect > closes WebSocket` | Clean disconnection |
| `disconnect > emits onDisconnected` | Disconnect event |
| `disconnect > emits onError` | Error event |
| `stale socket protection` | Ignores events from old socket after reconnect |
| `lastFinalProcMs reset` | Watermark resets on new connection |

## Acceptance criteria that could not be tested automatically

All acceptance criteria are covered by the tests above. The "connection lifecycle is clean" criterion is partially covered by the disconnect and stale socket tests; full leak detection (e.g., memory profiling) is out of scope for unit tests.

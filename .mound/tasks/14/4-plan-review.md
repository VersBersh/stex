**Plan Issues**
1. Critical — Step 3’s cursor-at-end detection is incorrect. In [3-plan.md](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/.mound/tasks/14/3-plan.md#L80), the plan compares `prevSelection.anchor.offset` to total document length, but Lexical anchor offsets are node-local, not document-global. That breaks as soon as the document has multiple text nodes, which the current append path already creates by appending a fresh `TextNode` for each token batch in [TokenCommitPlugin.tsx](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/src/renderer/overlay/editor/TokenCommitPlugin.tsx#L34). Fix: compute an absolute document offset for the selection, or detect “at end” by comparing against the actual last text node and its local end offset.

2. Critical — Steps 1 and 4 would break the existing inline-typing contract at the document tail. The plan routes all user text changes through `applyEdit()` in [3-plan.md](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/.mound/tasks/14/3-plan.md#L40) and [3-plan.md](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/.mound/tasks/14/3-plan.md#L157), but the spec requires tail typing to create or extend `source: "user"` blocks, not mark the prior `soniox` block modified. See [inline-typing.md](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/spec/features/inline-typing.md#L37) and current merge behavior in [editorBlockManager.ts](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/src/renderer/overlay/editor/editorBlockManager.ts#L16). Fix: split “mid-document edit” handling from “tail insertion” handling, with an explicit user-tail insertion path.

3. Major — Steps 1 and 4 are underspecified for edits that span block boundaries. The plan says multi-block edits should “mark all affected blocks as modified” in [3-plan.md](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/.mound/tasks/14/3-plan.md#L45), but it never defines how block texts are rewritten, truncated, merged, or removed so that `EditorBlockManager.getDocumentText()` stays equal to the Lexical document. That is a real gap because select+replace is explicitly in scope in [inline-editing.md](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/spec/features/inline-editing.md#L18). Fix: specify a concrete normalization algorithm for cross-block replacements, or explicitly narrow scope.

4. Major — Steps 7 and 8 assume test infrastructure that does not exist. The repo’s Vitest config only includes `src/**/*.test.ts` in [vitest.config.ts](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/vitest.config.ts#L3), there is no jsdom setup or React testing dependency in [package.json](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/package.json#L23), and the current editor tests are pure unit tests like [editorBlockManager.test.ts](/C:/code/draftable/stex/.mound/worktrees/worker-8-b756886e/src/renderer/overlay/editor/editorBlockManager.test.ts#L1). Fix: either add a prerequisite step for DOM/React test support, or extract the selection/diff logic into pure functions and test those directly.

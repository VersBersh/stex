# Test Notes

## Test files created

### 1. `src/renderer/overlay/editor/editorBlockManager.test.ts` (extended)

Added two new `describe` blocks to the existing test file:

- **`getSnapshot / restoreSnapshot`** — Tests the new snapshot/restore methods on EditorBlockManager:
  - Deep clone verification (mutation isolation)
  - Restore from snapshot
  - No aliasing between restored state and snapshot object

- **`undo/redo integration (snapshot-based)`** — Tests full undo/redo semantics at the block manager level:
  - Edit then undo restores `modified` flag to `false`
  - Edit → undo → redo round-trip restores `modified: true` and edited text
  - Multi-block snapshot capture and restore
  - Cross-block edit undo restores removed blocks
  - Tail typing undo removes user block

### 2. `src/renderer/overlay/editor/UndoRedoBlockSyncPlugin.test.ts` (new)

Tests the coordination logic between `blockHistory` stacks and `blockManager` — the same logic the plugin implements — without needing a Lexical instance:

- **PUSH detection**: Snapshot is pushed when a new undo entry is created
- **MERGE detection**: No additional snapshot on merged (same top) entries
- **Redo stack clearing**: New PUSH clears redo stack
- **Undo flow**: Pops from undo, pushes current to redo, restores snapshot
- **Redo flow**: Reverse of undo
- **No-op on empty stacks**: Undo/redo with empty stacks don't crash
- **Stack clearing (token commit)**: Both stacks are properly cleared
- **No stale snapshots**: After clear + new edits, undo restores to new state
- **Multiple undo/redo cycles**: Three edits → three undos → three redos

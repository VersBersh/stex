# Test Notes

## Test files created/modified

- `src/main/settings.test.ts` — extended existing file

## What each test covers

### `resolveSonioxApiKey` (existing, preserved)
- Saved value wins over env var
- Empty saved value falls back to env var
- Empty string when neither is set
- Whitespace-only saved value treated as non-empty

### `APP_SETTINGS_DEFAULTS`
- All keys present with correct default values matching task description
- Type-satisfies `AppSettings` (compile-time check)

### `getSettings`
- Returns defaults when store is empty
- Resolves `sonioxApiKey` via env var fallback
- Does NOT write resolved env var value back to the store (no writeback)
- Returns saved key when non-empty, ignoring env var

### `setSetting`
- Updates a single setting in the store
- Does not affect other settings

### `registerSettingsIpc`
- Registers handlers for both `SETTINGS_GET` and `SETTINGS_SET` channels
- GET handler returns current settings
- SET handler updates setting and broadcasts to all windows

## Acceptance criteria coverage

| Criterion | Test coverage |
|-----------|--------------|
| Settings persist across app restarts | Covered by testing `setSetting` stores to the mock (electron-store handles disk persistence) |
| Default values on first launch | `APP_SETTINGS_DEFAULTS` test + `getSettings` returns defaults |
| Readable/writable from renderer via IPC | `registerSettingsIpc` tests for GET and SET handlers |
| `settings:updated` broadcast on change | SET handler broadcast test |
| Types match `AppSettings` | Compile-time type check in defaults test |

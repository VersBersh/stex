# Test Notes

## Test files modified

- `src/main/settings.test.ts` — Added tests for encryption, masking, and renderer-safe settings

## What each test covers

### API key encryption (new suite)
- `setSetting encrypts sonioxApiKey before storing` — Verifies that the stored value is not plaintext
- `getSettings decrypts sonioxApiKey from store` — Round-trip: encrypt on write, decrypt on read
- `handles legacy plaintext values gracefully (migration)` — Simulates an existing plaintext value; expects it to be returned as-is
- `does not encrypt non-sonioxApiKey settings` — Verifies other settings are unaffected
- `handles empty sonioxApiKey without encryption` — Empty strings are not encrypted

### getSettingsForRenderer (new suite)
- `returns masked sonioxApiKey` — Verifies masked format with bullets and last 4 chars
- `returns empty string when no key is set` — No key = empty string
- `masks short keys (4 chars or less) without bullets` — Keys <=4 chars are shown fully
- `does not affect other settings fields` — Only sonioxApiKey is masked
- `masks env var fallback key` — Env var keys are also masked

### registerSettingsIpc (updated suite)
- `SETTINGS_GET handler returns masked sonioxApiKey` — IPC returns masked, not plaintext
- `SETTINGS_SET handler broadcasts masked sonioxApiKey` — Broadcast contains masked key

## Acceptance criteria not testable automatically
- Visual verification that the settings UI shows masked key and edit/display modes work correctly (requires manual Electron app testing)

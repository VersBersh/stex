## Acceptance criteria not tested automatically

- **Device list refreshes on add/remove**: The focus-based refresh in `index.tsx` adds a window event listener. Testing this would require jsdom window focus simulation which is fragile. The IPC layer (which provides fresh data on each call) is tested.
- **Selected device persists**: Already covered by existing `startCapture` tests that verify device name resolution from settings.
- **Selected device used for transcription**: Already covered by existing `startCapture` tests (resolves device by name, passes ID to PortAudio).

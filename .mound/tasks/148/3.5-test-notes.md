# Test Notes

## Test files created

- `src/main/audio-ring-buffer.test.ts`

## What each test covers

- **push**: `currentMs` advances correctly, `oldestMs` tracks first chunk, single-sample edge case
- **sliceFrom**: empty buffer → null, exact timestamp match, mid-chunk timestamp, latest chunk, evicted timestamp → null, chunk boundary matching
- **circular eviction**: capacity enforcement, `oldestMs` updates, evicted range returns null
- **clear**: resets `currentMs`, `oldestMs`, buffer contents; push works after clear
- **default capacity**: 5-minute default holds 300 seconds without eviction, evicts on overflow

## Acceptance criteria coverage

All acceptance criteria are covered by automated tests:
- `AudioRingBuffer` class with `push`, `sliceFrom`, `clear` — tested via all test groups
- Buffer capacity ~5 minutes — tested in "default capacity" group
- `push` advances monotonic session-time counter — tested in "push" group
- `sliceFrom(ms)` returns concatenated audio — tested in "sliceFrom" group
- Circular eviction — tested in "circular eviction" group
- `sliceFrom` returns null for evicted timestamps — tested in "sliceFrom" and "circular eviction" groups

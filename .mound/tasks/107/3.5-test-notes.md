# Test Notes

## Test files modified

### `src/main/logger.test.ts`
- `getLogFilePath` returns the log file path after successful initialization
- `getLogFilePath` returns null in console-only mode (failed init)

### `src/main/settings.test.ts`
- `registerSettingsIpc` registers handlers for `LOG_PATH_GET` and `LOG_REVEAL`
- `LOG_PATH_GET` handler returns the log file path from `getLogFilePath()`
- `LOG_PATH_GET` handler returns null when no log file
- `LOG_REVEAL` handler calls `shell.showItemInFolder` with the path
- `LOG_REVEAL` handler is a no-op when log path is null

### `src/preload/settings-preload.test.ts`
- `getLogPath` is exposed and calls `ipcRenderer.invoke` with `'log:get-path'`
- `revealLogFile` is exposed and calls `ipcRenderer.invoke` with `'log:reveal'`

## Acceptance criteria not covered by automated tests
- Visual rendering of the log path in the General settings page (requires a running Electron app or component tests with DOM rendering)
- "Show in folder" button actually opens the OS file manager (requires real Electron shell integration)

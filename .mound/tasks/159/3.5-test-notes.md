# Test Notes

## Test files created

- `src/main/permissions.test.ts` — Tests for the new permissions module

## What each test covers

### `permissions.test.ts`
- Verifies `initPermissions()` sets both `setPermissionCheckHandler` and `setPermissionRequestHandler` on the default session
- Permission check handler grants `media`, `clipboard-sanitized-write`, `clipboard-read`, and other permissions (all local content)
- Permission request handler grants `media` requests
- Permission request handler denies non-media requests

## Existing tests to update

- `src/main/first-run.test.ts` — Needs a mock for `./permissions` module since `initApp()` will now import and call `initPermissions()`. Also needs to verify `initPermissions` is called.
- `src/preload/settings-preload.test.ts` — The `getAudioDevices` function now returns `{ devices, permissionDenied }` instead of `string[]`. The existing test just checks the function exists, which will still pass.

## Acceptance criteria that could not be tested automatically

- "enumerateDevices() reliably returns device labels on fresh installs" — This depends on Electron's runtime behavior with the permission check handler and cannot be verified in unit tests with mocked Electron APIs. Requires manual testing or an Electron integration test.
- "Existing audio capture in the overlay window continues to work correctly" — Requires end-to-end testing with actual audio hardware.

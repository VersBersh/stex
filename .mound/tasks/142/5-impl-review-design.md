2. Major — **Hidden Coupling / error classification contract is broken**. [audio-capture.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/renderer/overlay/audio-capture.ts#L49) defines `normalizeAudioError`, but [OverlayContext.tsx](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/renderer/overlay/OverlayContext.tsx#L206) forwards raw `err.message` strings, and [error-classification.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/main/error-classification.ts#L3) still classifies by message text. Browser failures like `NotAllowedError` and `NotReadableError` will therefore surface as `unknown` instead of `mic-denied` / `mic-unavailable`, so the user loses the intended remediation path. Normalize errors at the capture boundary and send only normalized messages to main, or move classification to something stronger than free-form strings.

3. Major — **Semantic Coupling / settings contract is violated**. In [audio-capture.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/renderer/overlay/audio-capture.ts#L72), if the saved `deviceName` no longer matches an enumerated device, capture silently falls back to the default microphone. That is a bad hidden side effect: the app can record from the wrong device while the settings still claim a specific input is selected. Fail fast with a normalized “Audio device not found” error instead of silently changing devices.

4. Major — **Temporal Coupling / tail audio is dropped on stop**. The worklet batches 1600 samples and only posts full buffers in [audio-capture.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/renderer/overlay/audio-capture.ts#L18), but teardown in [audio-capture.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/renderer/overlay/audio-capture.ts#L117) never flushes the partially filled batch. That means pause/stop/finalize can discard up to ~100 ms of trailing speech, which is exactly where users often stop speaking. Add an explicit flush handshake before disconnecting, or redesign batching so shutdown cannot lose buffered PCM.

5. Minor — **DRY / architecture clarity**. Device enumeration logic is duplicated between [audio-capture.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/renderer/overlay/audio-capture.ts#L141) and [settings-preload.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/preload/settings-preload.ts#L20), while the exported helper in `audio-capture.ts` is currently unused. That gives the browser-device policy two reasons to change and makes drift likely. Extract one shared browser-side helper and have both call sites use it.

6. Minor — **Testing gap on the riskiest path**. [settings-preload.test.ts](/C:/code/draftable/stex/.mound/worktrees/worker-1-300188a2/src/preload/settings-preload.test.ts#L65) now only checks that `getAudioDevices` exists; it no longer verifies permission-denied handling, filtering, or even that `navigator.mediaDevices` is used correctly. Given the move from IPC to direct browser APIs, that leaves the new behavior effectively unprotected. Add focused tests around `getUserMedia` failure, empty labels, and device-name filtering.

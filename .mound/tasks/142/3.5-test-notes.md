# Test Notes

## Test files created/modified

### `src/main/audio.test.ts` (rewritten)
- `startCapture` sends `AUDIO_START_CAPTURE` to renderer with device name from settings
- `startCapture` sends null device when `audioInputDevice` is null
- `startCapture` throws if already capturing
- `stopCapture` sends `AUDIO_STOP_CAPTURE` to renderer
- `stopCapture` is a no-op when not capturing
- `stopCapture` allows restart after stopping
- `registerAudioIpc` registers handlers for `AUDIO_CHUNK` and `AUDIO_CAPTURE_ERROR`
- `AUDIO_CHUNK` handler calls `onData` callback with Buffer
- `AUDIO_CHUNK` handler is a no-op when not capturing
- `AUDIO_CAPTURE_ERROR` handler sends stop to renderer before calling `onError`
- `AUDIO_CAPTURE_ERROR` handler clears capturing state (can restart)
- `AUDIO_CAPTURE_ERROR` handler is a no-op when not capturing

### `src/preload/index.test.ts` (to be updated during implementation)
- New methods: `onAudioStartCapture`, `onAudioStopCapture`, `sendAudioChunk`, `sendAudioCaptureError`

### `src/preload/settings-preload.test.ts` (to be updated during implementation)
- `getAudioDevices` test updated to reflect it's now a direct renderer function (no IPC invoke)

## Acceptance criteria not automatically testable

- **Bluetooth headset compatibility** — requires physical Jabra Evolve2 65 hardware test
- **Latency** — subjective, requires manual verification
- **Volume meter display** — requires running the full Electron app
- **Device selection UI** — requires running the settings window

# Test Notes

## Test files created

- `src/renderer/overlay/editor/analyzeReplayEligibility.test.ts`

## What each test covers

1. **No dirty nodes** — all clean TimestampedTextNodes → `eligible: false`, `blockedReason: 'none'`
2. **Eligible with clean tail** — dirty node near end with clean tail in same paragraph → `eligible: true`
3. **Proximity gate** — dirty node >100 chars from end → `blockedReason: 'too-far-from-end'`
4. **No clean tail** — dirty node at doc end → `blockedReason: 'dirty-tail'`
5. **Paragraph boundary** — dirty node and clean tail in different paragraphs → `blockedReason: 'paragraph-boundary'`
6. **Suffix-match replayStartMs** — `currentText.endsWith(originalText)` → uses dirty node's `startMs`
7. **Non-suffix-match replayStartMs** — no suffix match → uses first clean node's `startMs`
8. **Plain TextNode as dirty** — user-typed content detected as dirty
9. **Proximity boundary exact** — 100 chars = eligible, 101 chars = not
10. **Empty editor** — `eligible: false`, `blockedReason: 'none'`
11. **All clean (suffix-match edge)** — clean nodes aren't selected as dirty
12. **Multiple dirty nodes** — picks the latest dirty node

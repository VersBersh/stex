In this app, closing all windows does not quit the process, so a missing/bad icon can leave the app running with no visible tray affordance and no obvious quit path. The risk is worse because `git status` shows `resources/` as untracked, so the reviewed diff does not actually carry the asset it now requires. Fix: include the `.ico` in the committed change set, and make icon loading fail-safe by either aborting startup with a clear error or falling back to a known-good embedded/default icon instead of continuing with an empty image.

A reader has to know that `app.getAppPath()` and `electron-builder.json.files` must stay aligned for the icon lookup to work.

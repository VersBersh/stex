## What each test covers

### `mergeTokens`
- Single word as pending (not yet complete)
- Multi-word batch emits completed words, buffers last
- Sub-word chunks merged within a batch (`"Hell"` + `"o,"` + `" world"`)
- Cross-batch word continuation (batch 1 partial → batch 2 continues)
- First token without leading space treated as word start
- Multi-word batch with sub-word splits (realistic data from debug-seed.json)
- Empty incoming with no pending
- Preserves pending when incoming is empty
- Filters out empty-text tokens
- Punctuation-only continuation tokens (`.` after word)
- Question mark continuation (`?` after `"you"`)

### `flushPending`
- Returns null for empty pending
- Converts single pending token to merged word
- Merges multiple pending tokens into one word
- Filters empty-text tokens in pending
- Returns null when all pending tokens have empty text

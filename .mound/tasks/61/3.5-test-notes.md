# Test Notes

## Test files created/modified
- `src/renderer/overlay/editor/ghost-text.test.ts` — added `createGhostTextController` tests

## What each test covers
- **sets --ghost-text-content on non-final tokens**: Basic ghost text display
- **concatenates multiple non-final tokens**: Multi-token ghost text
- **removes on empty non-final tokens**: Clearing when no non-final data
- **removes on empty string token text**: Edge case for empty text tokens
- **clears ghost text on final tokens**: Core acceptance criterion — ghost text cleared when finals arrive
- **allows new ghost text after final tokens clear it**: Recovery after clearing
- **full handoff sequence**: End-to-end non-final → final → non-final lifecycle
- **no-op when getRoot returns null**: Null safety (editor not mounted)
- **handleFinalTokens idempotent**: Double-clear safety

## Untestable acceptance criteria
- "Visually seamless transition" — requires visual/integration testing with actual browser rendering, not achievable in the current Node-only Vitest setup

# Test Notes

No new test files created.

## Rationale

The new `lexicalTextContract.ts` module exports:
1. `LEXICAL_PARAGRAPH_SEPARATOR` — a string constant (`'\n\n'`). No logic to test.
2. `$getDocumentText()` — a one-line wrapper around `$getRoot().getTextContent()`. No branching, no logic.

The existing test files already cover the paragraph separator behavior:
- `InlineEditPlugin.test.ts` — tests `findTextDiff` with `\n\n` separator strings
- `editorBlockManager.test.ts` — tests block manager operations with `\n\n` separator strings

These tests will break if the contract changes, which is the desired behavior. Adding a unit test for `$getDocumentText()` would require mocking Lexical's `$getRoot`, providing no value beyond confirming the mock works.

## Verification approach

Instead of new tests, the migration will be verified by:
1. Grepping that no direct `$getRoot().getTextContent()` calls remain in the three migrated plugins
2. Running the existing test suite to confirm nothing is broken

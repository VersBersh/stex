# Test Notes

## Test files created

### `src/main/session.test.ts` (modified)
Added `describe('API key guard')` block with 5 tests:
- Shows overlay and sends no-api-key error when API key is empty
- Does not start session when API key is empty
- Sends open-settings action in no-api-key error
- Clears error when starting session with valid API key
- Hides overlay on second toggle when shown without API key

### `src/main/first-run.test.ts` (new)
Tests for the extracted `initApp()` function:
- Opens settings when API key is empty
- Does not open settings when API key exists
- Initializes all managers regardless of API key state
- Initializes all managers when API key exists
